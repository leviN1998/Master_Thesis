## 🚀 Overview

Estimating the spin of a table tennis ball is crucial for robotic table tennis systems. Traditional frame-based cameras suffer from motion blur at high spin rates and short observation windows.

This project introduces an **event-camera-based learning pipeline** that:

* captures asynchronous event streams
* converts them into voxel representations
* processes them with a ConvGRU network
* predicts spin direction and speed

The system is trained primarily on **synthetic trajectories** generated by a custom Blender-based simulator and later fine-tuned on real recordings.

## 📊 Results

| Task         | Performance                   |
| ------------ | ----------------------------- |
| 2-class spin | **~98–99% accuracy**          |
| 6-class spin | **~96–97% accuracy**          |
| Regression   | **~12.7° mean angular error** |

> Models trained on improved synthetic data generalize significantly better to real event recordings.

## 🧪 Data Pipeline

### Event preprocessing

* spike-based shot extraction
* circular kernel ball tracking
* 100×100 ROI cropping
* voxel grid encoding

### Synthetic data

* Blender-based trajectory generation
* physics-aware spin sampling
* realistic lighting calibration
* event camera simulation
